 # obsidian-anki-agent-skills

Два скилла для ИИ-агента Claude Code, которыми я
пользуюсь каждый день для собственной учёбы (Python, SQL, английский):

- **anki-cards** — агент читает мои заметки, сам выбирает, что из материала
  действительно стоит запомнить, составляет карточки и добавляет их в Anki.
- **obsidian-organize** — приводит сырой текст к моему формату конспектов
  в Obsidian и помогает держать заметки в порядке.

Это не разовые промпты, а готовые инструменты с прописанным порядком работы.

## Демо

Короткое видео: агент собирает HTML-квиз из заметки Obsidian и по его итогам создает карточки в Anki.

## Что такое «скилл» для агента

Скилл — это инструкция для агента (файл `SKILL.md`), рядом с ней
вспомогательные скрипты и шаблоны. Агент сам понимает по запросу
пользователя, какой скилл подключить, и дальше действует по шагам,
описанным в инструкции. То, что должно работать одинаково каждый раз
(добавление карточек в Anki, запуск квиза), вынесено в небольшие Python-
скрипты. Так у агента получается предсказуемый и повторяемый порядок
работы, а не каждый раз чуть разное поведение.

## Скилл 1: anki-cards

Делает карточки для Anki из учебного материала.

Главная идея — **не нарезать материал механически**. Агент идёт по пяти
шагам и на каждом решает, что вообще достойно карточки:

1. Читает заметку и понимает, о чём она.
2. Прикидывает, насколько пользователь знаком с темой (чаще всего это видно
   из самой заметки, лишний раз не переспрашивает).
3. Отделяет то, что заметка объясняет, от фона, и отбрасывает очевидное.
4. Показывает список кандидатов в карточки с объяснением, что берёт и что
   пропускает, — пользователь может вернуть пропущенное.
5. Составляет карточки по правилам качества и добавляет их в Anki.

**Режим HTML-квиза (по желанию).** Вместо того чтобы делать карточки по всему
материалу, агент сначала собирает короткий тест в виде локальной HTML-страницы
и открывает её в браузере. Пользователь проходит квиз — и карточки создаются
только по тем вопросам, где он ошибся или ответил «не знаю». Так повторение
бьёт по реальным пробелам, а не по тому, что человек и так помнит.

Что ещё продумано внутри:
- перед добавлением агент проверяет, нет ли уже похожей карточки, чтобы не
  плодить дубли;
- есть отдельный файл с правилами качества и примерами «плохо → хорошо»
  ([`card-quality.md`](skills/anki-cards/references/card-quality.md)):
  один факт на карточку, понятный вопрос, короткий конкретный ответ.

## Скилл 2: obsidian-organize

Берёт сырой текст (статью, конспект лекции, черновик) и приводит его к моему
личному формату заметок в Obsidian: заголовки, определения, блоки кода,
теги по смыслу. Кроме этого умеет:

- собирать MOC-хаб (карту раздела) — заметку-оглавление со ссылками на материалы
  темы, сгруппированными по смыслу;
- наводить порядок: находить одинокие заметки, дубли, повторяющиеся имена и
  ссылки, ведущие в никуда.

Важное правило: **агент ничего не записывает и не удаляет без явного
подтверждения**. Vault (хранилище Obsidian-заметок) живой и хранится в
iCloud, поэтому агент сначала показывает результат целиком и ждёт ответа, и
только потом что-то меняет.

## Что внутри репозитория

```
skills/
├── anki-cards/
│   ├── SKILL.md               # инструкция для агента
│   ├── references/
│   │   └── card-quality.md    # правила и примеры хороших карточек
│   ├── scripts/
│   │   ├── import_cards.py    # добавление карточек в Anki
│   │   └── quiz_server.py     # запуск квиза в браузере
│   └── templates/
│       └── quiz_template.html # шаблон страницы с тестом
└── obsidian-organize/
    └── SKILL.md                # инструкция для агента
```

## Что нужно для запуска

- **Python 3** — скрипты используют только стандартную библиотеку, ничего
  доустанавливать не нужно.
- **Anki** с дополнением
  [AnkiConnect](https://ankiweb.net/shared/info/2055492159) — через него
  агент добавляет карточки.
- **Obsidian** — обычные markdown-файлы, никаких особых плагинов.

Путь к Vault в инструкциях указан как `~/path/to/your/vault` — нужно
подставить свой.
